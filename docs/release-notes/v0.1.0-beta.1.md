## Proxmox VE Mobile v0.1.0-beta.1

First public beta for Android users testing mobile Proxmox VE workflows. This release is intended for real-world feedback on login, TLS handling, infrastructure inspection, VM/LXC lifecycle actions, and task follow-up. It is not a full desktop web UI replacement yet.

## Install

- Download the signed APK attached to this GitHub Release.
- The signed APK is the public beta build. Debug APKs and unsigned workflow artifacts are for development/testing only.
- Android may require enabling installation from the browser or file manager used to open the APK.

## Included In This Beta

- Password and API token login.
- TLS validation with SHA-256 fingerprint pinning support for self-signed environments.
- Dashboard, node, VM, LXC, task, storage, network, user, backup, and cluster inspection flows.
- VM/LXC lifecycle actions with confirmation, task notices, and direct task-detail follow-up.
- Read-only visibility for VM/LXC details, snapshots, selected configuration data, task history/logs, storage content, network interfaces, users, backups, and cluster status.
- Android CI coverage for unit tests, lint, debug/release builds, instrumentation compilation, and release-like TLS smoke compilation.

## Known Limitations

- Guest console access is outside this beta scope.
- VM/LXC configuration editing is outside this beta scope.
- Snapshot create, delete, and rollback are outside this beta scope.
- Backup create, restore, download, and delete are outside this beta scope.
- User create, edit, and delete flows are outside this beta scope.
- Node reboot, shutdown, shell, and advanced service actions are outside this beta scope.
- Storage, network, user, backup, and cluster areas are primarily read-only.
- TFA-specific password login handling is not complete; API token login is the preferred beta workaround where appropriate.
- Play Store and F-Droid packaging are deferred while GitHub Releases distribution is validated.

## Verified Smoke Coverage

- First-beta readiness gate passed with no pending, failed, or blocked release checks.
- Dashboard, node detail, VM list/detail, and LXC list routing were exercised on a Pixel-class Android emulator.
- Login rendering, API-token controls, certificate fingerprint validation, and release-like TLS guardrails are covered by instrumentation smoke tests.
- Route state restoration and fake-backed navigation coverage exist for dashboard, node, VM, LXC, task, storage, network, users, backups, cluster, and settings flows.
- The signed release workflow and Android CI both passed for this tag.

## Report Issues

Please include Android version, Proxmox VE major version, auth method, TLS setup, and sanitized logs or screenshots. Remove hostnames, usernames, IP addresses, node names, VM/container names, tokens, cookies, tickets, and certificate fingerprints before posting public issue details.
