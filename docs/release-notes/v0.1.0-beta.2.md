## Proxmox VE Mobile v0.1.0-beta.2

Patch beta for the first public Android release. This update fixes a login connection issue where a LAN Proxmox host could accidentally reach Android's cleartext HTTP policy instead of using HTTPS, and adds a safer first-use trust flow for self-signed Proxmox certificates.

## Install

- Download the signed APK attached to this GitHub Release.
- This APK replaces `v0.1.0-beta.1` for beta testing.
- For a standard Proxmox VE host, enter the host or IP with port `8006`, or paste a full HTTPS URL.
- If Android does not trust the Proxmox certificate, review the certificate dialog and trust the fingerprint only if it is the expected server.

## Fixed

- Bare host/IP input now normalizes through the HTTPS Proxmox default instead of producing a raw Android cleartext-policy error.
- Pasted HTTPS server URLs are normalized before authentication and saved credential use.
- Self-signed Proxmox certificates can now be reviewed and trusted on first use; accepted certificates are pinned by SHA-256 fingerprint for future logins.
- Non-local HTTP login attempts are blocked before network access with guidance to use HTTPS on port 8006.
- Localhost and Android emulator loopback HTTP remain available for development smoke tests only.
- Failed pre-network validation now clears any previous authenticated session state.

## Still In Beta

- Guest console access, VM/LXC configuration editing, snapshot mutation, backup mutation, user mutation, node power actions, and Proxmox Backup Server management remain outside this beta scope.
- Storage, network, user, backup, and cluster areas are primarily read-only.
- TFA-specific password login handling is not complete; API token login is the preferred beta workaround where appropriate.

## Report Issues

Please include Android version, Proxmox VE major version, auth method, TLS setup, and sanitized logs or screenshots. Remove hostnames, usernames, IP addresses, node names, VM/container names, tokens, cookies, tickets, and certificate fingerprints before posting public issue details.
